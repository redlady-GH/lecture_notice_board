# [리서치] 교육용 프로젝트 고도화 및 차기 커리큘럼 제안

## 1. 개요
본 문서는 **"Lecture Notice Board"** 프로젝트(Level 1~2)를 성공적으로 마친 학습자들을 위해, **최신 웹 개발 트렌드**와 **심화 보안 기술**을 반영한 차기 커리큘럼(Level 3~4)의 방향성을 제안합니다.

현재 프로젝트가 "**기본기(Fundamentals)**"와 "**배포(Deployment)**"에 집중했다면, 다음 단계는 "**확장성(Scalability)**", "**모던 스택(Modern Stack)**", 그리고 "**자동화(Automation)**"가 핵심 키워드가 되어야 합니다.

---

## 2. Flask 모던 스택 도입 (Code Modernization)

2024-2025년의 Python 웹 생태계는 "**비동기(Async)**"와 "**타입 안정성(Type Safety)**"을 중시하는 방향으로 진화하고 있습니다.

### 2.1. SQLAlchemy 2.0 도입 (ORM)
*   **현황**: 현재 `sqlite3` 라이브러리를 사용한 Raw SQL 방식. SQL 이해에는 좋으나 생산성과 유지보수성이 떨어짐.
*   **제안**: **SQLAlchemy 2.0** 스타일의 ORM 도입.
    *   `Model` 클래스 정의를 통한 스키마 관리.
    *   `Alembic`을 이용한 데이터베이스 마이그레이션(Migration) 실습.
    *   **교육적 가치**: 객체 지향 프로그래밍(OOP)과 관계형 데이터베이스(RDBMS)의 매핑 원리 학습.

### 2.2. 구조적 리팩토링 (Blueprints & Application Factory)
*   **현황**: `app.py` 단일 파일에 모든 로직이 집중된 모놀리식 구조.
*   **제안**: **Flask Blueprint**를 활용한 모듈화.
    *   `auth`, `board`, `admin` 등으로 기능을 분리.
    *   `create_app()` 팩토리 패턴 적용으로 테스트 용이성 확보.
    *   **교육적 가치**: 대규모 프로젝트를 관리하는 아키텍처 설계 능력 배양.

### 2.3. 데이터 검증 (Pydantic)
*   **현황**: 수동으로 `if not request.form['title']:` 같은 검증 로직 작성.
*   **제안**: **Pydantic** 도입.
    *   입력 데이터의 타입 검증과 에러 처리를 자동화.
    *   **교육적 가치**: 견고한(Robust) 애플리케이션을 만드는 방어적 프로그래밍 기법 학습.

## 3. 보안 심화 (Advanced Security)

기초적인 CSRF/XSS 방어를 넘어, 실제 서비스 수준의 보안 아키텍처를 구축합니다.

### 3.1. 인증 고도화 (OAuth 2.0 & OIDC)
*   **현황**: 단일 관리자 비밀번호 사용.
*   **제안**: **OAuth 2.0**을 이용한 소셜 로그인(Google, GitHub) 구현.
    *   `Authlib` 라이브러리 활용.
    *   JWT(JSON Web Token)와 Session 방식의 차이점 비교 학습.
    *   **교육적 가치**: 현대 웹 서비스의 표준 인증 체계 이해.

### 3.2. 공격 방어 (Rate Limiting & CSP)
*   **현황**: 무차별 대입 공격(Brute Force)에 대한 방어책 부재.
*   **제안**:
    *   **Flask-Limiter**: IP 기반 요청 제한(Rate Limiting) 구현.
    *   **CSP (Content Security Policy)**: HTTP 헤더를 통해 XSS를 원천 차단하는 브라우저 보안 정책 적용.
    *   **교육적 가치**: 능동적인 보안 방어 체계 구축 경험.

## 4. 인프라 및 DevOps (Infrastructure Evolution)

"내 컴퓨터에서만 되는 코드"를 넘어, 팀 단위 협업과 자동화를 지향합니다.

### 4.1. CI/CD 파이프라인 구축
*   **현황**: 수동으로 `docker-compose up` 실행.
*   **제안**: **GitHub Actions** 도입.
    *   코드 푸시 시 자동 린트(Lint) 및 테스트(Test) 수행.
    *   Docker Image 자동 빌드 및 레지스트리(Docker Hub) 푸시.
    *   **교육적 가치**: 현대적인 소프트웨어 배포 수명주기(SDLC) 체험.

### 4.2. 데이터베이스 확장 (PostgreSQL)
*   **현황**: 파일 기반 SQLite 사용 (동시성 한계).
*   **제안**: **PostgreSQL** 컨테이너 도입.
    *   `docker-compose`에 DB 서비스 추가 및 네트워크 연동.
    *   데이터 백업 및 복구 전략 수립.
    *   **교육적 가치**: 엔터프라이즈급 RDBMS 운영 경험.

## 5. 프론트엔드 트렌드 (Frontend Modernization)

React/Vue 같은 SPA(Single Page Application)는 학습 곡선이 높습니다. Flask 개발자에게 적합한 대안을 제시합니다.

### 5.1. HTMX 도입 (Server-Driven UI)
*   **현황**: 페이지 이동 시 전체 새로고침 발생.
*   **제안**: **HTMX** 라이브러리 활용.
    *   HTML 속성만으로 AJAX 요청과 부분 렌더링 구현.
    *   JavaScript 작성 없이도 SPA와 유사한 사용자 경험 제공.
    *   **교육적 가치**: "복잡한 JS 프레임워크 없이도 모던한 웹을 만들 수 있다"는 대안적 시각 제공.

## 6. 제안 로드맵 (Recommended Roadmap)

| 단계 | 주제 | 핵심 기술 | 목표 |
| :--- | :--- | :--- | :--- |
| **Level 3** | **모던 아키텍처** | Blueprint, SQLAlchemy, Pydantic | 유지보수 가능한 구조로 리팩토링 |
| **Level 4** | **보안 및 인증** | OAuth 2.0, Rate Limiting, HTTPS(Let's Encrypt) | 실제 서비스 수준의 보안 구축 |
| **Level 5** | **DevOps & Auto** | GitHub Actions, PostgreSQL, Pytest | 자동화된 테스트 및 배포 파이프라인 |

이 로드맵은 학생들이 "**단순 코더**"에서 "**아키텍트**"로 성장하는 데 필요한 기술적 깊이를 제공할 것입니다.
