# 프로젝트 분석 및 평가 보고서

## 1. 개요
본 보고서는 **"Flask와 SQLite를 활용한 프로젝트 일정 관리 보드"** 프로젝트의 코드, 아키텍처, 문서를 분석하고, **초급 개발자 양성과정 4개월 차** 교육생을 대상으로 한 교육 자료로서의 적합성을 평가합니다.

## 2. 프로젝트 구성 분석

### 2.1. 기술 스택
- **Language**: Python 3.9
- **Web Framework**: Flask
- **Database**: SQLite (Raw SQL 사용)
- **Infrastructure**: Docker, Docker Compose
- **Web Server/WAS**: Nginx, Gunicorn

### 2.2. 코드 구조 (`app.py`)
- **패턴**: 단일 파일(`app.py`)에 라우팅, DB 연결, 비즈니스 로직이 모두 포함된 **Monolithic 구조**입니다.
- **DB 접근**: ORM(SQLAlchemy)을 사용하지 않고 `sqlite3` 라이브러리를 통해 **SQL을 직접 작성**하는 방식을 채택했습니다.
- **보안**: `python-dotenv`를 도입하여 비밀번호와 시크릿 키를 환경 변수로 분리했습니다.

### 2.3. 인프라 아키텍처
- **컨테이너화**: `Dockerfile`을 통해 애플리케이션 실행 환경을 코드화했습니다.
- **배포 구조**: `Nginx (SSL/Proxy) -> Gunicorn (WSGI) -> Flask`로 이어지는 **표준적인 프로덕션 배포 구조**를 갖추고 있습니다.
- **보안 통신**: Self-Signed Certificate를 이용한 **HTTPS**를 적용하여 내부망 통신 보안을 강화했습니다.
- **네트워크**: mDNS(`*.local`)를 지원하여 유동 IP 환경에서도 고정된 도메인으로 접근 가능합니다.
- **운영 안정성**: `restart: always` 정책을 적용하여 서버 재부팅 시 서비스가 자동 복구되도록 설정되었습니다.
- **데이터 영속성**: Docker Volume을 사용하여 컨테이너 재시작 시에도 데이터가 유지되도록 구성되었습니다.

## 3. 교육적 적합성 평가 (대상: 4개월 차 초급 개발자)

### 3.1. 난이도: 적절함 (★★★★☆)
- **Python/Flask**: 4개월 차 학생이 이해하기에 적절한 수준입니다. 복잡한 클래스 구조나 데코레이터 남용 없이 직관적인 함수형 뷰로 작성되었습니다.
- **SQL**: ORM 대신 Raw SQL을 사용한 점은 **SQL 기초를 다지기에 매우 훌륭한 선택**입니다. 학생들은 쿼리가 어떻게 동작하는지 명확히 이해할 수 있습니다.
- **Docker & Network**: HTTPS 설정과 mDNS 구성은 다소 난이도가 있으나, **실무 네트워크 환경**을 이해하는 데 큰 도움이 됩니다.

### 3.2. 구현 수준: 실무 지향적 기초 (★★★★☆)
- 단순한 `python app.py` 실행을 넘어, **Nginx와 Gunicorn을 연동**하고 **HTTPS를 적용**한 점이 돋보입니다.
- 환경 변수(`.env`) 사용과 CSRF 보호 적용은 보안 의식을 고취시키는 좋은 실습 포인트입니다.

### 3.3. 개선이 필요한 교육 포인트 (Weakness & Opportunities)
- **비밀번호 저장**: 현재 비밀번호가 평문으로 비교되고 있습니다. 이는 교육용이라도 **해싱(Hashing)** 의 필요성을 언급하거나, `werkzeug.security`를 적용하는 과제로 확장하기 좋습니다.
- **구조 확장**: 모든 로직이 `app.py`에 있어 코드가 길어질 수 있습니다. 향후 **Flask Blueprint**를 사용하여 라우트를 분리하는 리팩토링 수업으로 연결할 수 있습니다.

## 4. 종합 평가 및 제언

### 4.1. 총평
이 프로젝트는 **"웹 개발의 A to Z"를 콤팩트하게 경험**할 수 있는 훌륭한 교보재입니다. 데이터베이스 연동부터 웹 서버 구성, 도커 배포, 그리고 **HTTPS 보안 설정**까지의 흐름이 매끄럽게 연결되어 있습니다.

### 4.2. 향후 발전 과제 (학생들에게 제시할 수 있는 도전 과제)
1.  **보안 강화**: 관리자 비밀번호를 DB에 해싱하여 저장하고 검증하는 기능 구현.
2.  **공인 배포**: Let's Encrypt를 사용하여 공인 인증서를 발급받고 실제 도메인 연결하기.
3.  **UI 개선**: Bootstrap 등을 적용하여 모바일 반응형 디자인 적용.
4.  **CI/CD 맛보기**: GitHub Actions를 통해 Docker Image를 자동으로 빌드해보기.

## 5. 최근 개선 사항 (Recent Improvements)
*   **HTTPS 적용**: Nginx에 SSL 설정을 추가하여 통신 암호화 구현.
*   **mDNS 지원**: 내부망 도메인 접속 편의성 증대.
*   **CSRF 보호**: 게시글 삭제 기능에 CSRF Token 검증 로직 추가 및 Method를 POST로 변경하여 보안 취약점 해결.
*   **Hot Reload**: 개발 편의성을 위해 Docker Compose에 볼륨 마운트 및 Gunicorn Reload 옵션 추가.

## 6. 결론
**"배포 가능한 최소 기능 제품"**(MVP)으로서의 완성도가 높으며, 4개월 차 개발자가 **자신의 코드가 실제 서버에서 어떻게 돌아가는지** 큰 그림을 그리기에 아주 적합한 프로젝트입니다.
